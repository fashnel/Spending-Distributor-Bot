

# 🤖 Spending-Distributor-Bot

Телеграм-бот для автоматизированного учета расходов и распределения их по категориям в Google Таблицах. 

> **Note**: Логика бота и структура запросов были реализованы с помощью **Qwen CLI**.

## 🛠 Стек технологий
* **Language:** Python 3.12
* **Database/Storage:** Google Sheets API
* **Deployment:** Docker & Docker Compose
* **CI/CD:** GitHub Actions + Self-hosted Runner

## 🚀 DevOps & Инфраструктура
В ходе работы над этим проектом я значительно прокачал свои навыки в автоматизации и развертывании:

* **Контейнеризация (Docker):** Весь проект полностью упакован в Docker.
* **Оркестрация (Docker Compose):** Я осознал, насколько удобен compose для управления жизненным циклом приложения. Одной командой поднимается всё окружение, прокидываются переменные и монтируются волюмы.
* **CI/CD (GitHub Actions):** * Настроен пайплайн автоматической сборки и пуша Docker-образов в **GitHub Container Registry (GHCR)**.
    * Реализован автоматический деплой при каждом пуше в ветку `main`.
* **Self-hosted Runners:** Так как у меня нет "белого" IP от провайдера, я развернул и настроил раннер агента от GitHub внутри локальной сети. Это позволило организовать полноценный CI/CD цикл без необходимости пробрасывать SSH наружу.

## 📊 Функционал таблиц
Бот не просто кидает данные в таблицу, а соблюдает строгую структуру:
- **Локализация:** Числа записываются с запятыми (формат `24,00`) для корректной работы в RU-локалях.
- **Динамические формулы:** При создании нового листа автоматически генерируется строка "Итого", которая динамически рассчитывает сумму по всем категориям (колонки B, C, D...) с помощью формул `=СУММ()`.

## 📦 Как запустить

1. Клонируйте репозиторий:
   ```bash
   git clone [https://github.com/ваше_имя/Spending-Distributor-Bot.git](https://github.com/ваше_имя/Spending-Distributor-Bot.git)
   ```
2. Создайте файл .env на основе .env.example и заполните своими данными.
3. * Перейдите в Google Cloud Console.
   * Создайте проект и включите Google Sheets API и Google Drive API.
   * В разделе Credentials создайте «Service Account».
   * Создайте JSON-ключ для этого аккаунта, скачайте его и переименуйте в client_secret.json в корне проекта.
4. Запустите проект:
   ```bash
   docker compose up -d
   ```
